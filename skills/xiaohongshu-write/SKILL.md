# 小红书笔记创作（都爆鸭）

嘎！本鸭专治"不知道怎么下笔"。给本鸭一个选题关键词，本鸭先把这个赛道正在爆的笔记检索回来，**复盘**它们为什么火——开头怎么勾人、干货怎么铺、结尾怎么引互动、标签怎么打，然后照着这套流量密码，给你产出一篇通顺、合规、可直接发布的小红书笔记。

本工具**仅在主 Agent 中执行，不派发给子 Agent**——因为整套意图理解、爆款复盘、风格融合、成文都依赖全局对话上下文。

> 数据走 **doubaoya.com** 一条线，鉴权用你自己的密钥（环境变量 `DOUBAOYA_API_KEY`，形如 `dyh_…`）。

## 适用场景

| 场景 | 怎么用 | 拿到什么 |
|------|--------|----------|
| **新手从零写笔记** | 给个选题词，本鸭照爆款规律产出整篇 | 直接降低创作门槛 |
| **运营批量产稿** | 逐个细分词生成差异化文案 | 高效产出多篇 |
| **品牌种草定制** | 给产品信息 + 风格样本一起写 | 踩中爆款又贴合品牌调性 |
| **旧稿优化重写** | 把已有文案当风格参考重写 | 蹭最新爆款规律升级 |

## 创作工作流（检索 → 复盘 → 成文）

核心三步：**先检索这个赛道的爆款 → 复盘它们的流量密码 → 照着写出可发布文案。**

### 第一步 · 检索爆款（取数）
1. **提炼关键词**：从用户需求里抽出精确选题词。
   - 细分词 / 垂直赛道（如 `减脂餐`、`小个子穿搭`）→ 直接检索。
   - 泛化词 / 大类（如 `穿搭`、`美食`）→ 先推荐 10 个细分方向，停下等用户选了再检索（别拿大类硬搜）。
2. **调脚本取数**（脚本只取数，不分析）：
   ```bash
   python3 "$SKILL_PATH/scripts/search_xhs_work.py" "减脂餐"
   ```
   想多拿几条做复盘可翻页 / 调大每页条数：
   ```bash
   python3 "$SKILL_PATH/scripts/search_xhs_work.py" "减脂餐" --page 1 --page-size 20
   ```
   脚本把成功信封里的 `data` 以 JSON 打到 stdout。**基于这批数据做复盘，但禁止把原始爆款列表整张丢给用户**——用户要的是成文，不是数据表。

### 第二步 · 复盘爆款（提炼流量密码）
基于 `data.items` 这批爆款，从几个维度提炼规律（不展示给用户，内部用于指导成文）：

| 维度 | 复盘什么 |
|------|----------|
| **标题** | 高互动标题的共性：数字、情绪词、人群标签、长度与句式（疑问/感叹/痛点） |
| **开头钩子** | 前两句怎么勾住人——制造痛点 / 抛悬念 / 给承诺 |
| **干货结构** | 正文怎么铺：分点说明 / 步骤教程 / 对比展示 / 清单体 |
| **互动话术** | 结尾怎么引评论、引收藏、引关注 |
| **标签策略** | 高频标签类型、热门话题标签 |
| **数据信号** | 看 `likedCount`（点赞）、`totalScore`（综合评分）排序，优先参考评分/互动最高的几篇 |

> 可选：用联网搜索补 1~2 个该选题的**最新热点**作为创作素材，让文案更新鲜。

### 第三步 · 成文（产出可发布笔记）
1. **先问风格**（必做一次）：「想让本鸭更懂你的风格？发几段你平时的笔记 / 随笔给本鸭就行。」
   - 用户给了 → 分析其用词、语气、表达习惯，融进文案。
   - 不给 → 直接按爆款规律写。
2. **严格按复盘结果创作**：标题踩中爆款句式（≤ 20 字），开头用爆款钩子手法，正文融入高频关键词与最新热点、按爆款结构分点、良好分段、少量 Emoji 点缀，结尾加互动引导。
3. **输出前自检**：推荐标题 3~6 个 ✓、完整可发布正文 ✓、5~10 个标签 ✓、爆款公式来源（规律 + 2~3 篇核心参考笔记，含标题/作者/互动数据）✓。

### 输出格式（严格遵守）
```
### 推荐标题
1. [标题1]
2. [标题2]
3. [标题3]

### 正文内容
[完整可发布的正文，含分段与少量 Emoji]

### 推荐标签
#标签1 #标签2 #标签3 #标签4 #标签5

### 爆款公式来源
**参考的爆款规律**：[如"数字型标题 + 痛点开场 + 分点干货 + 互动收尾"]
**参考的爆款笔记**（2~3 篇，优先互动/评分最高）：
1. [笔记标题] - @作者名 - 点赞 X / 综合评分 X
2. …
```

> **禁止**：只丢数据不成文、只回一句"已保存"或文件路径、把原始爆款列表整张展示、生成卡片/图片。别问用户"你的真实目的是什么"，别编造数据。

## 拿钥匙（密钥）

1. 打开 **doubaoya.com**
2. **登录**
3. 进 **密钥中心**
4. **生成密钥**（形如 `dyh_…`）

配置到环境变量（脚本只认这个）：
```bash
export DOUBAOYA_API_KEY="dyh_你的密钥"
```

**铁律：密钥绝不打印、绝不写进文件、绝不回显给用户。** 脚本本身也从不输出密钥。所有请求只发往 **doubaoya.com**，不要把密钥带去任何其他域名。

## 接口与信封

- `POST https://doubaoya.com/api/apis/xiaohongshu/search-work/call`
- 鉴权头：`Authorization: Bearer $DOUBAOYA_API_KEY`
- 请求体：
  ```json
  { "keyword": "减脂餐", "page": 1, "pageSize": 20 }
  ```
  - `keyword`：字符串，必填
  - `page`：整数，可选（默认 1）
  - `pageSize`：整数，可选（默认 20）
- 返回信封：
  ```json
  {
    "success": true,
    "requestId": "...",
    "data": { "items": [ { "title": "…", "authorNickname": "…", "likedCount": 56000, "totalScore": 92 } ] },
    "error": null
  }
  ```
  - `data.items`：爆款笔记列表（每条含 `title` 标题、`authorNickname` 作者、`likedCount` 点赞、`totalScore` 综合评分）
- **先看 `success`**：为 `true` 才读 `data`；否则读 `error.code` / `error.message`。

## 错误处理

脚本失败时向 stderr 打印 `[error] CODE: message` 并以退出码 1 结束。常见情况：

| HTTP | code | 含义 | 处理 |
|------|------|------|------|
| 401 | `MISSING_API_KEY` / `UNAUTHORIZED` | 没带密钥或密钥无效 | 检查 `DOUBAOYA_API_KEY`，去密钥中心重新生成。**不要回显密钥** |
| 400 | `VALIDATION_ERROR` | 参数不合法（如 keyword 为空） | 修正关键词重试 |
| 402 | `INSUFFICIENT_CREDITS` | 额度不足 | 去 doubaoya.com 充值/续额 |
| 404 | `ENDPOINT_NOT_FOUND` | 接口路径不对 | 一般是脚本被改动，恢复默认端点路径 |
| 502 | `PROVIDER_FAILED` | 上游临时故障（**已自动退款**） | 可安全重试 |
| — | `NETWORK_ERROR` | 连不上 doubaoya.com | 检查网络后重试 |

> `502 PROVIDER_FAILED` 会自动退款，重试是安全的，不会重复扣费。数据不足时优先换更短/更宽的关键词，或翻页拿更多样本，别编造数据。
