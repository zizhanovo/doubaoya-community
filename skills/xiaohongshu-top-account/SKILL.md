# 小红书最夯账号榜 · 都爆鸭

嘎！本鸭帮你把小红书上最夯的账号捞成一张榜——日榜、周榜、月榜随你挑，赛道随你切，
排名、粉丝数、互动一行行铺在终端里，看完再给你点一句"这榜该怎么用来对标"。

> 数据走 **doubaoya.com** 一条线，鉴权用你自己的口令（环境变量 `DOUBAOYA_API_KEY`，形如 `dyh_…`）。

---

## 适用场景

| 场景 | 怎么用 | 拿到什么 |
|------|--------|----------|
| **头部账号对标** | 查某赛道日/周/月榜，看头部都是谁 | 一眼锁定该赛道的标杆账号 |
| **竞品跟踪** | 周期性看榜，盯竞品名次涨跌 | 摸清对手在赛道里的位置变化 |
| **起号找标杆** | 看自己赛道里谁在涨，作为追赶目标 | 给新号定一个可参照的对标对象 |
| **达人初筛 / 选号** | 按赛道拉榜，按粉丝量级圈候选 | 快速给投放/合作筛一批候选账号 |
| **赛道热度研究** | 横向比不同赛道的头部体量 | 判断哪个赛道更卷、机会在哪 |

## 工作流（4 步）

### 1. 读懂意图，定清三个参数
从用户描述里确定：
- **周期 `--period`**：日榜 `day` / 周榜 `week` / 月榜 `month`（默认 `day`）。用户说"这周""本月"就切对应周期。
- **赛道 `--type`**：默认 `综合全部`。用户说了具体方向就映射到标准赛道名（见下表）。
- **日期 `--rank-date`**：默认昨天（当日数据通常尚未结算），一般不用动；用户要查历史某天再传。

**赛道（type）参考清单**（口语词 → 标准赛道名）：

| 用户说 | 映射到 |
|--------|--------|
| 不限 / 全部 | 综合全部 |
| 美食、探店、烹饪、做饭 | 美味佳肴 |
| 美妆、彩妆、护肤 | 化妆美容 |
| 个护、洗护 | 个人护理 |
| 穿搭、时尚 | 时尚穿搭 |
| 旅行、旅游 | 旅行度假 |
| 母婴、育儿 | 亲子育儿 |
| 健身、运动、瑜伽 | 体育锻炼 |
| 数码、科技 | 数码科技 |
| 家居、装修 | 居家装修 |
| 学习、教育 | 学习教育 |
| 宠物 | 宠物天地 |

> 完整赛道清单（可直接用）：综合全部、出行代步、休闲爱好、影视娱乐、数码科技、医疗保健、综合杂项、星座情感、时尚穿搭、婚庆婚礼、拍摄记录、学习教育、化妆美容、居家装修、旅行度假、亲子育儿、个人护理、美味佳肴、职业发展、宠物天地、潮流鞋包、日常生活、科学探索、新闻资讯、体育锻炼。

### 2. 调用榜单脚本
```bash
python3 "$SKILL_PATH/scripts/fetch_xhs_top_account.py"
```
切周期 + 赛道：
```bash
python3 "$SKILL_PATH/scripts/fetch_xhs_top_account.py" --period week --type 美味佳肴
```
查历史某天：
```bash
python3 "$SKILL_PATH/scripts/fetch_xhs_top_account.py" --period month --rank-date 2026-06-01
```
脚本把成功信封里的 `data` 以 JSON 打到 stdout。**每次查询只跑一次脚本**，直接读完整 stdout，别用 `head`/`tail` 预览或重复调用。

> 若某赛道某天返回空，多半是当日榜单尚未结算——可把 `--rank-date` 往前挪一两天再试。

### 3. 渲染榜单表格
从 `data.items`（账号数组）里取字段，按下面格式铺成 Markdown 表格。字段做防御式读取——`rank`（排名）/ `nickname`（博主）/ `fansCount`（粉丝数）/ 互动相关字段可能缺失，缺了就留空或标"暂无"，别报错、别补值。博主名可渲染成指向其小红书主页的可点链接（若返回里带主页 URL/ID）。

| 排名 | 博主 | 粉丝数 | 互动 |
|------|------|--------|------|
| 1 | 示例博主 | 128.0w | 32.1w |

> 粉丝数 / 互动做千分位或"万"档可读化；想加"综合指数"列按热度排序时，那是展示层加工，不影响接口调用。

### 4. 给一句对标洞察
表格之后，用本鸭的口吻补一句**对标洞察**：这个赛道的头部体量大概在什么档、谁在快速上位、哪几个账号最值得当起号标杆。简短、有据，别堆套话。

## 拿钥匙（口令）

1. 打开 **doubaoya.com**
2. **登录**
3. 进 **口令中心**
4. **生成口令**（形如 `dyh_…`）

配置到环境变量（脚本只认这个）：
```bash
export DOUBAOYA_API_KEY="dyh_你的口令"
```

**铁律：口令绝不打印、绝不写进文件、绝不回显给用户。** 脚本本身也从不输出口令。所有请求只发往 **doubaoya.com**，不要把口令带去任何其他域名。

---

## 接口与信封

- `POST https://doubaoya.com/api/apis/xiaohongshu/xiaohongshu-top-account/call`
- 鉴权头：`Authorization: Bearer $DOUBAOYA_API_KEY`
- 请求体：`{ "period": "day", "rankDate": "2026-06-26", "type": "综合全部" }`
  - `period`：榜单周期，`day` / `week` / `month`（默认 `day`）
  - `rankDate`：榜单日期 `YYYY-MM-DD`（默认昨天）
  - `type`：赛道/分类（默认 `综合全部`）
- 返回信封：
  ```json
  {
    "success": true,
    "requestId": "...",
    "data": { "items": [ { "rank": 1, "nickname": "...", "fansCount": 1280000 } ] },
    "error": null
  }
  ```
- **先看 `success`**：为 `true` 才读 `data`；否则读 `error.code` / `error.message`。

## 错误处理

脚本失败时向 stderr 打印 `[error] CODE: message` 并以退出码 1 结束。常见情况：

| HTTP | code | 含义 | 处理 |
|------|------|------|------|
| 401 | `MISSING_API_KEY` / `UNAUTHORIZED` | 没带口令或口令无效 | 检查 `DOUBAOYA_API_KEY`，去口令中心重新生成 |
| 400 | `VALIDATION_ERROR` | 参数不合法（如赛道名/日期格式有误） | 核对 `--type` / `--rank-date` 重试 |
| 402 | `INSUFFICIENT_CREDITS` | 额度不足 | 去 doubaoya.com 充值/续额 |
| 502 | `PROVIDER_FAILED` | 上游临时故障（**已自动退款**） | 可安全重试 |

> `502 PROVIDER_FAILED` 会自动退款，重试是安全的，不会重复扣费。
