# 公众号写作助手（都爆鸭）

写公众号别凭空硬憋。本鸭的打法是**先看市场再下笔**：用你的主题把近期同主题的**爆文**拉回来当样本，然后带主 Agent 从这批真跑量的文章里反推——标题怎么起、选题切哪个口、正文怎么搭，最后落到一篇能发的稿。脚本只负责把爆文样本取回来；提炼与成文，由主 Agent 在样本上做。

> 数据走 **doubaoya.com** 一条线，鉴权用你自己的口令（环境变量 `DOUBAOYA_API_KEY`，形如 `dyh_…`）。
>
> 只想看爆文榜不写作，用 `wechat-hot-article`（同一接口、定位是"搜"）；本鸭定位是"写"——多走一步把样本变成稿。

---

## 适用场景

| 场景 | 怎么用 | 拿到什么 |
|------|--------|----------|
| **起标题** | 拉同主题爆文，反推高打开标题套路 | 几个可直接用的标题方向 |
| **找选题切口** | 看这批爆文都从哪个角度切 | 一个还没写透的选题角度 |
| **正文搭结构** | 拆头部爆文的开头/论证/收尾骨架 | 一套可复用的正文结构 |
| **参考仿写** | 以爆款为样本写同主题新稿 | 一篇能发的完整文章 |
| **内容选品** | 看哪个子主题已经跑出量 | 判断这个主题值不值得写 |

## 工作流（4 步）

### 1. 读懂写作意图，提炼主题词
先弄清用户**想写什么、写给谁、想达到什么目的**，再抽出**主题关键词**（精简 2~6 字，如 `AI 工具`、`职场沟通`）。一次只拉一个主题。

### 2. 拉同主题爆文样本
```bash
# 默认：最近 7 天的同主题爆文（end=今天，start=今天往前 6 天）
python3 "$SKILL_PATH/scripts/hot_write.py" "AI 工具"

# 指定时间区间
python3 "$SKILL_PATH/scripts/hot_write.py" "AI 工具" --start 2026-06-01 --end 2026-06-20
```
脚本把成功信封里的 `data`（这批爆文样本）以 JSON 打到 stdout。**只跑一次**，直接读完整 stdout。

### 3. 从样本提炼写作要点
在样本（`data.items`）上做提炼，重点三件：

- **标题套路**：头部几篇标题的共性结构（数字 / 反差 / 痛点 / 悬念 / 身份代入），抽 2~3 个可用框架。
- **选题切口**：这批爆文从哪个角度切；哪个角度密度高（已拥挤）、哪个还稀疏（可抢）。
- **正文结构**：拆 1~2 篇头部爆文的骨架——怎么开头钩人、中段怎么论证 / 举例、结尾怎么收 / 引导。

字段做**防御式读取**，缺失就跳过，别报错。

### 4. 产出标题候选 + 选题角度 + 正文骨架（按需成稿）
给用户：3~5 个标题候选 + 1~2 个推荐选题角度 + 一份正文结构骨架。用户要完整稿时，再按选定的标题与骨架在样本风格上写出整篇。全程**以样本为据**，别脱离数据空谈套路。

## 拿钥匙（口令）

1. 打开 **doubaoya.com**
2. **登录**
3. 进 **口令中心**
4. **生成口令**（形如 `dyh_…`）

配进环境变量（脚本只认这个）：
```bash
export DOUBAOYA_API_KEY="dyh_你的口令"
```

**铁律：口令绝不打印、绝不写进文件、绝不回显给用户。** 脚本本身也从不输出口令。所有请求只发往 **doubaoya.com**。

---

## 接口与信封

- `POST https://doubaoya.com/api/apis/gongzhonghao/hot-article/call`
- 鉴权头：`Authorization: Bearer $DOUBAOYA_API_KEY`
- 请求体：`{ "keyword": "AI 工具", "startDate": "2026-06-21", "endDate": "2026-06-27" }`
  - `keyword`：字符串，必填（写作主题词）
  - `startDate` / `endDate`：`YYYY-MM-DD`。脚本默认 `endDate=今天`、`startDate=今天往前 6 天`，可用 `--start` / `--end` 覆盖。
- 返回信封：
  ```json
  {
    "success": true,
    "requestId": "...",
    "data": { "items": [ { "title": "...", "accountName": "...", "readCount": 0, "publishTime": "..." } ] },
    "error": null
  }
  ```
- **先看 `success`**：为 `true` 才读 `data`；否则读 `error.code` / `error.message`。

## 错误处理

脚本失败时向 stderr 打印 `[error] CODE: message` 并以退出码 1 结束。

| HTTP | code | 含义 | 处理 |
|------|------|------|------|
| 401 | `MISSING_API_KEY` / `UNAUTHORIZED` | 没带口令或口令无效 | 检查 `DOUBAOYA_API_KEY`，去口令中心重新生成 |
| 400 | `VALIDATION_ERROR` | 参数不合法（如主题词为空、日期格式错） | 修正后重试 |
| 402 | `INSUFFICIENT_CREDITS` | 额度不足 | 去 doubaoya.com 充值/续额 |
| 502 | `PROVIDER_FAILED` | 上游临时故障（**已自动退款**） | 可安全重试 |

> `502 PROVIDER_FAILED` 会自动退款，重试安全，不会重复扣费。
