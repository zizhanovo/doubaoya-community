# 小红书文旅笔记内容源（都爆鸭）

本鸭帮你按关键词扫一遍小红书里的文旅爆款笔记——把城市攻略、景区打卡、出片机位里赞藏跑得最高的笔记捞出来，终端铺成可点的爆款表格，再帮你聚类成几个文旅选题方向，攒成一份能直接套用的种草对标清单。

> 小红书是文旅种草的第一入口：一篇攻略、一组机位就能决定游客去哪。适合**文旅局、景区运营、城市文旅账号、旅游博主**找种草选题、看爆款笔记、抠标题与封面切口。

> 数据走 **doubaoya.com** 一条线，鉴权用你自己的口令（环境变量 `DOUBAOYA_API_KEY`，形如 `dyh_…`）。

> 说明：底层是小红书内容流 Feed，按你给的关键词取笔记。「文旅」是用关键词把它当文旅选题镜头来用——结果取决于 Feed 对你这个词的命中，词越贴近文旅越准。

## 适用场景

| 场景 | 怎么用 | 拿到什么 |
|------|--------|----------|
| **种草选题** | 用 `文旅`、`旅游攻略` 扫高赞笔记 | 看清文旅笔记当下种什么火 |

| **爆款对标** | 看高赞笔记集中在哪些角度 | 找到同赛道值得对标的内容角度 |
| **标题切口拆解** | 看高赞笔记的标题与钩子 | 提炼文旅笔记的爆款标题套路 |

## 工作流（4 步）

### 1. 定关键词
从用户描述里抽出一个 **文旅方向关键词**（必填），如 `文旅`、`旅游攻略`、`景区打卡`、`出片机位`、`小众目的地`。词宽命中多、词细更聚焦，一次只扫一个关键词。

### 2. 调用脚本
```bash
python3 "$SKILL_PATH/scripts/fetch_culture_feed.py" "旅游攻略"
```
翻页 / 调整每页条数（可选）：
```bash
python3 "$SKILL_PATH/scripts/fetch_culture_feed.py" "景区打卡" --page 2 --size 30
```
脚本把成功信封里的 `data` 以 JSON 打到 stdout。**每次查询只跑一次脚本**，直接读完整 stdout，别用 `head`/`tail` 预览，也别对同一参数重复调用。

### 3. 渲染爆款表格
从 `data.items`（笔记数组）里取字段，铺成 Markdown 表格。本接口稳定返回的核心字段是 `title`（标题）与 `likeCount`（点赞）；做防御式读取——字段可能缺失，缺了就留空，别报错，也别脑补接口没返回的链接或作者。若某条返回里恰好带了链接，再把标题渲染成可点链接，否则纯文本即可。

| 标题 | 点赞 |
|------|------|
| 示例文旅笔记 | 8,800 |

### 4. 聚类成选题方向 + 一句洞察
把这批笔记按文旅角度**聚类**（如 城市攻略 / 景区打卡 / 出片机位 / 美食探店 / 周末短途），每个方向挑出代表爆款，组成一份简短对标清单。结尾用本鸭口吻补一句**选题洞察**：当下文旅笔记哪个角度最种草、哪种封面/标题还有空位。别堆套话，别反问用户目的。

## 拿钥匙（口令）

1. 打开 **doubaoya.com**
2. **登录**
3. 进 **口令中心**
4. **生成口令**（形如 `dyh_…`）

配置到环境变量（脚本只认这个）：
```bash
export DOUBAOYA_API_KEY="dyh_你的口令"
```

**铁律：口令绝不打印、绝不写进文件、绝不回显给用户。** 脚本本身也从不输出口令。所有请求只发往 **doubaoya.com**。

---

## 接口与信封

- `POST https://doubaoya.com/api/apis/xiaohongshu/xiaohongshu-playlet-feed/call`
- 鉴权头：`Authorization: Bearer $DOUBAOYA_API_KEY`
- 请求体：`{ "keyword": "旅游攻略", "pageNum": 1, "pageSize": 20 }`
  - `keyword`：字符串，必填
  - `pageNum`：整数，可选（默认 1，由 `--page` 控制）
  - `pageSize`：整数，可选（默认 20，由 `--size` 控制）
- 返回信封：
  ```json
  {
    "success": true,
    "requestId": "...",
    "data": { "items": [ { "title": "...", "likeCount": 8800 } ] },
    "error": null
  }
  ```
- **先看 `success`**：为 `true` 才读 `data`；否则读 `error.code` / `error.message`。

## 错误处理

脚本失败时向 stderr 打印 `[error] CODE: message` 并以退出码 1 结束。常见情况：

| HTTP | code | 含义 | 处理 |
|------|------|------|------|
| 401 | `MISSING_API_KEY` / `UNAUTHORIZED` | 没带口令或口令无效 | 检查 `DOUBAOYA_API_KEY`，去口令中心重新生成 |
| 400 | `VALIDATION_ERROR` | 参数不合法（如 keyword 为空） | 修正关键词重试 |
| 402 | `INSUFFICIENT_CREDITS` | 额度不足 | 去 doubaoya.com 充值/续额 |
| 502 | `PROVIDER_FAILED` | 上游临时故障（**已自动退款**） | 可安全重试 |

> `502 PROVIDER_FAILED` 会自动退款，重试是安全的，不会重复扣费。
