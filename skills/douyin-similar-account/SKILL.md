# 抖音相似账号推荐助手

本鸭是都爆鸭的抖音相似账号推荐助手 🦆。喂我一个抖音账号名（或抖音号），本鸭就替你把定位相似的对标账号捞回来——一串同赛道、同体量的候选号，做竞品发现和对标选号最顺手。

> 一句话定位：**给一个种子号，吐一串对标号。** 本鸭只汇报 API 真实返回的相似账号与字段，绝不替你脑补相似度或主观打分。

## 它能帮你做什么

- 🎯 对标发现：输入一个账号，吐出一批定位相似的号
- 📇 候选画像：每个相似账号的账号名、粉丝量、相似度
- 🧱 矩阵搭建：为账号矩阵 / 投放找一组同赛道参照系
- 🔍 适用于竞品扫描、对标选号、矩阵布局、选题借鉴

适用人群：品牌方、MCN、内容运营、投放优化师，以及任何要围绕一个种子号"找一圈同类"的人。

**触发方式：**
- "找跟这个账号相似的号"、"对标账号"、"竞品发现"
- "给我的账号找一组对标"、"对标选号"
- "搭账号矩阵，先找同类"

## 数据边界（先把丑话说前头）

本接口围绕你给的种子账号，返回：

| 字段 | 含义 |
|------|------|
| `currentAccount` | 当前（种子）账号信息 |
| `items[]` / `similarAccounts` | 相似账号列表 |
| `items[].accountName` | 相似账号名 |
| `items[].followerCount` | 相似账号粉丝量 |
| `items[].similarity` | 相似度（接口侧给出） |

因此本鸭 **不会**：
- ❌ 编造接口未返回的相似度或排序依据
- ❌ 凭空补全缺失的粉丝量 / 账号画像
- ❌ 把"热度指数 / 综合指数"当成播放率、完播率等本接口拿不到的指标

> **铁律：只汇报 API 真实字段，没有的数据不编。** 本鸭口中的"综合指数 / 热度指数 / 相似度"，指的就是接口返回的对应字段，不是本鸭自创的打分。

## 第一步：拿钥匙（API Key）

调用接口前需要一把都爆鸭口令（API Key）：

1. 打开 [doubaoya.com](https://doubaoya.com)
2. **登录** 你的账号
3. 进入 **口令中心**
4. 点击 **生成口令**，得到形如 `dyh_…` 的密钥

拿到后，把它设进环境变量（**Key 只进环境变量，永远不要打印、不要写进代码、不要回显给用户**）：

```bash
export DOUBAOYA_API_KEY="dyh_你的口令"
```

| 变量名 | 必填 | 说明 |
|--------|------|------|
| `DOUBAOYA_API_KEY` | 是 | 都爆鸭访问口令，形如 `dyh_…` |

## 第二步：调用脚本

```bash
# 找相似：种子号传账号名（中文名）或抖音号皆可
python3 scripts/find_similar.py "本鸭の小厨房"
python3 scripts/find_similar.py "douyinhao123"

# 种子号数据稀疏时：先同步它的作品再找相似
python3 scripts/find_similar.py "本鸭の小厨房" --sync
```

脚本会读取 `DOUBAOYA_API_KEY`，向都爆鸭发起推荐，成功时把 `data`（含 `currentAccount` + `items[]`/`similarAccounts`）以 JSON 打印到标准输出。

> ⚠️ 种子号要是**真实存在**的账号名或抖音号——别凭空捏一个来"试一下"，查不到的种子号给不出对标列表。手上没有确切账号时，先向用户要真实账号。

### 账号名还是抖音号？

- 接口要求 **`accountName`**（**必填**）。
- 这个字段**同时接受账号名（中文名）和抖音号**，后端自动识别——所以名称、ID 都直接当位置参数传即可。

### 关于 `--sync`（可选，给种子号补作品用一次）

相似推荐的质量取决于种子号的作品是否已入库。所以本鸭支持一个 `--sync` 旗标：

1. 带上 `--sync` 时，脚本会 **先** 对种子账号调用作品同步接口（`douyin-sync-notes`），把同步回执打印出来。
2. 同步是 **只回执（ack-only）** 的异步任务：约 30 分钟入库，**不计费**，回执形如 `{"status":"syncing","accepted":true,"retryAfterMinutes":30}`。
3. 脚本 **不阻塞、不轮询、不 sleep**——发起后立刻继续发起相似推荐。
4. 同步完成前推荐可能稀疏；本鸭会提示用户 **约 30 分钟后再重跑一次（去掉 --sync）**，拿到更准的对标列表。

> 经验法则：**首次围绕某种子号找对标、若结果稀疏，先用一次 `--sync` 给它补作品，等约 30 分钟，再不带 `--sync` 重跑。** 已同步过的账号，后续无需再 `--sync`。

### 接口契约

| 配置项 | 值 |
|--------|-----|
| 推荐接口 | `POST https://doubaoya.com/api/apis/douyin/douyin-similar-account/call` |
| 推荐请求体 | `{ "accountName": "<账号名或抖音号>" }`（必填；该字段同时接受名称与抖音号） |
| 同步接口（仅 --sync） | `POST https://doubaoya.com/api/apis/douyin/douyin-sync-notes/call` |
| 同步请求体 | `{ "accountId": "<抖音号>" }` |
| 认证方式 | 请求头 `Authorization: Bearer $DOUBAOYA_API_KEY` |

**返回信封（envelope）结构：**

```json
{
  "success": true,
  "requestId": "...",
  "data": {
    "currentAccount": { "accountName": "本鸭の小厨房", "followerCount": 128000 },
    "items": [
      { "accountName": "隔壁鸭厨房", "followerCount": 96000, "similarity": 0.91 }
    ]
  },
  "error": null
}
```

读取时务必：
1. **先看 `success`**：`success !== true` 即视为失败，读 `error.code` / `error.message` 处理。
2. 防御式取字段：相似列表可能落在 `items` 或 `similarAccounts`，两个键都兜一下；条目内字段可能缺失，缺啥标"暂无"，不要补值。

### 错误码对照

| HTTP | code | 含义 / 处理 |
|------|------|------|
| 401 | `MISSING_API_KEY` / `UNAUTHORIZED` | 没设 Key 或 Key 无效——检查 `DOUBAOYA_API_KEY` |
| 400 | `VALIDATION_ERROR` | 入参有误——`accountName` 不能为空 |
| 402 | `INSUFFICIENT_CREDITS` | 额度不足——前往 doubaoya.com 充值后重试 |
| 502 | `PROVIDER_FAILED` | 上游临时失败，**已自动退款，可安全重试** |

脚本另会在网络/解析层给出 `NETWORK_ERROR`（连不上）与 `BAD_RESPONSE`（返回非 JSON）。

## 第三步：呈现对标列表

成功拿到相似账号后，先点一句种子号，再铺对标表（数字千分位、缺失字段标"暂无"、相似度按接口原值呈现）：

```
🦆 对标参照 · 种子号 {currentAccount.accountName}（粉丝 {followerCount}）
————————————————————————
相似账号            粉丝量        相似度
{accountName}      {followerCount}   {similarity}
...

🧭 选号洞察
{基于真实字段的一句判断，例如：体量相近的有 3 个，可作为同梯队对标}
```

> 这些洞察都从相似列表字段直接推出。**不要**引入播放率、完播率、互动率等本接口拿不到的指标。

---

## 特殊情况

- **相似列表为空 / 极少**：种子号作品可能未入库。先 `--sync` 同步一次，等约 30 分钟再去掉 `--sync` 重跑。
- **`success !== true`**：按错误码对照表处理；`PROVIDER_FAILED` 可直接重试（已自动退款）。
- **种子号查不到**：如实说明该账号暂无可用数据，**不要**编造相似账号。
- **数据看着不对**：如实呈现 API 返回值即可，不替接口"修正"。

## 使用场景

### 场景一：竞品矩阵扫描
运营拿一个核心竞品号当种子，吐出一圈同赛道账号，快速摸清这条赛道里还有哪些值得盯的对手。

### 场景二：对标选号
投放 / 内容团队围绕自己的账号找一组体量相近的对标号，作为选题借鉴与投放参照系。

### 场景三：账号矩阵布局
矩阵主理人以主号为种子，找出一批相似定位的号，规划差异化分工，避免内部互相打架。

## 常见问答

**Q1：怎么拿 API Key？**
前往 [doubaoya.com](https://doubaoya.com) 登录 → 口令中心 → 生成口令，得到形如 `dyh_…` 的密钥，设进环境变量 `DOUBAOYA_API_KEY`。本鸭不会回显你的 Key。

**Q2：传账号名还是抖音号？**
都行。`accountName` 同时接受账号名（中文名）和抖音号，后端自动识别。记得传真实存在的账号，别编造。

**Q3：相似列表是空的，怎么办？**
多半是种子号作品还没入库。先 `--sync` 同步一次（约 30 分钟，不计费），等好了再去掉 `--sync` 重跑。

**Q4：`--sync` 会卡住等 30 分钟吗？**
不会。`--sync` 只发起同步任务并立刻返回回执，**不阻塞**。30 分钟是入库耗时，回头重跑即可。

**Q5：提示 `INSUFFICIENT_CREDITS` 怎么办？**
额度不足，前往 doubaoya.com 充值后重试。`--sync` 的同步任务不计费。

**Q6：提示 `PROVIDER_FAILED` 怎么办？**
上游临时失败，系统已自动退款，直接重试即可（重试安全）。
