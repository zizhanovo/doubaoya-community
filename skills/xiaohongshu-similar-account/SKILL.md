# 小红书对标账号推荐 · 都爆鸭

嘎！本鸭帮你找对标——给一个小红书号，或者说清"哪个赛道、多少粉、什么等级"，
本鸭就给你两组账号：**可直接抄玩法的同阶对标**，和**可追赶的高阶标杆**，
账号、粉丝数、相似度排成表，省你一个个翻主页猜。

> ⚠️ 本鸭只查本数据库里的对标账号，**不联网搜索**、不替你脑补主页内容——表里的每条都是接口真实返回。
> 数据走 **doubaoya.com** 一条线，鉴权用你自己的口令（环境变量 `DOUBAOYA_API_KEY`，形如 `dyh_…`）。

---

## 适用场景

| 场景 | 怎么用 | 拿到什么 |
|------|--------|----------|
| **起号找参考** | 给自己的 redId 或"赛道+粉丝量" | 一批可直接抄玩法的同阶账号 |
| **找追赶标杆** | 同上 | 粉丝量更高一档的高阶标杆，定方向 |
| **投放 / 合作选号** | 按赛道 + 粉丝量级筛 | 圈一批量级匹配的候选账号 |
| **竞品圈定** | 给竞品 redId 查相似 | 摸清同一身位的对手有哪些 |

适用人群：小红书创作者、KOL 运营、品牌方、MCN 机构。

## 两种输入模式

### 模式 A：按小红书号（redId）
用户给了具体账号（"我的 ID 是 27493135897""帮我找和 xxx 像的号"）→ 用 `--red-id` 精准查。

### 模式 B：按 赛道 + 粉丝量 + 账号等级
用户只描述了画像（"做饭的，3000 粉，素人"）→ 用 `--track` + 粉丝量 + `--level` 筛。

**至少要有 `--red-id` 或 `--track` 其一**，两者都没有时脚本会报错退出（请向用户追问）。

**粉丝量处理**：
- 单个数值（"3000 粉"）→ `--max-fans 3000 --min-fans 0`。
- 区间（"1000-3000 粉"）→ `--min-fans 1000 --max-fans 3000`。
- 没指定 → 两个都不传。

**赛道（track）参考**（口语词 → 标准赛道名，与"最夯账号榜"同一套）：

| 用户说 | 映射到 |
|--------|--------|
| 做饭、美食、探店 | 美味佳肴 |
| 美妆、护肤 | 化妆美容 |
| 穿搭、时尚 | 时尚穿搭 |
| 旅行、旅游 | 旅行度假 |
| 母婴、育儿 | 亲子育儿 |
| 健身、运动 | 体育锻炼 |
| 数码、科技 | 数码科技 |

> 完整赛道清单：综合全部、出行代步、休闲爱好、影视娱乐、数码科技、医疗保健、综合杂项、星座情感、时尚穿搭、婚庆婚礼、拍摄记录、学习教育、化妆美容、居家装修、旅行度假、亲子育儿、个人护理、美味佳肴、职业发展、宠物天地、潮流鞋包、日常生活、科学探索、新闻资讯、体育锻炼。
> 账号等级（level）常用档：素人、初级达人、腰部达人、头部达人 等（按用户口语映射，如"小白"→素人）。

## 工作流（3 步）

### 1. 调用推荐脚本
模式 A（按 redId）：
```bash
python3 "$SKILL_PATH/scripts/similar_xhs_account.py" --red-id 27493135897
```
模式 B（按赛道 + 粉丝量 + 等级）：
```bash
python3 "$SKILL_PATH/scripts/similar_xhs_account.py" --track 美味佳肴 --min-fans 0 --max-fans 3000 --level 素人
```
脚本把成功信封里的 `data` 以 JSON 打到 stdout。**每次只跑一次脚本**，读完整 stdout，别用 `head`/`tail` 预览。

### 2. 渲染对标表格
从 `data.items`（账号数组）里取字段，按下面格式铺成 Markdown 表格。字段做防御式读取——`nickname`（账号）/ `fansCount`（粉丝数）/ `similarity`（相似度）可能缺失，缺了留空，别报错、别补值。账号名可渲染成指向其小红书主页的可点链接（若返回里带主页 URL/ID）。

| 账号 | 粉丝数 | 相似度 |
|------|--------|--------|
| [示例账号](https://www.xiaohongshu.com/user/profile/xxx) | 2.8w | 0.92 |

> 若返回里能区分"同阶对标"和"高阶标杆"两组，可分两张表展示。每条后面可附一句"可学之处"，但只能基于返回数据总结，**不要联网去查它的主页内容**。

### 3. 给一句对标洞察
表格之后，用本鸭口吻补一句：哪个号最值得先抄、哪个标杆是合适的追赶目标。简短、有据，不联网、不脑补。

## 拿钥匙（口令）

1. 打开 **doubaoya.com**
2. **登录**
3. 进 **口令中心**
4. **生成口令**（形如 `dyh_…`）

配置到环境变量（脚本只认这个）：
```bash
export DOUBAOYA_API_KEY="dyh_你的口令"
```

**铁律：口令绝不打印、绝不写进文件、绝不回显给用户。** 脚本本身也从不输出口令。所有请求只发往 **doubaoya.com**，不要把口令带去任何其他域名。

---

## 接口与信封

- `POST https://doubaoya.com/api/apis/xiaohongshu/xiaohongshu-similar-account/call`
- 鉴权头：`Authorization: Bearer $DOUBAOYA_API_KEY`
- 请求体（只带提供了的字段）：
  - `redId`：小红书号（模式 A）
  - `track`：赛道（模式 B）
  - `minFans` / `maxFans`：粉丝量下/上限（整数，可为 0）
  - `level`：账号等级
  - 例：`{ "track": "美味佳肴", "minFans": 0, "maxFans": 3000, "level": "素人" }`
- 返回信封：
  ```json
  {
    "success": true,
    "requestId": "...",
    "data": { "items": [ { "nickname": "...", "fansCount": 28000, "similarity": 0.92 } ] },
    "error": null
  }
  ```
- **先看 `success`**：为 `true` 才读 `data`；否则读 `error.code` / `error.message`。

## 错误处理

脚本失败时向 stderr 打印 `[error] CODE: message` 并以退出码 1 结束。常见情况：

| HTTP | code | 含义 | 处理 |
|------|------|------|------|
| —    | （本地校验） | 既没给 `--red-id` 也没给 `--track` | 补上其一再跑 |
| 401 | `MISSING_API_KEY` / `UNAUTHORIZED` | 没带口令或口令无效 | 检查 `DOUBAOYA_API_KEY`，去口令中心重新生成 |
| 400 | `VALIDATION_ERROR` | 参数不合法（如赛道名/等级有误） | 核对 `--track` / `--level` 重试 |
| 402 | `INSUFFICIENT_CREDITS` | 额度不足 | 去 doubaoya.com 充值/续额 |
| 502 | `PROVIDER_FAILED` | 上游临时故障（**已自动退款**） | 可安全重试 |

> `502 PROVIDER_FAILED` 会自动退款，重试是安全的，不会重复扣费。
